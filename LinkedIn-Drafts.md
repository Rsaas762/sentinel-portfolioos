### Post 2 — Sentinel WebGuard
> Most small businesses have no idea whether their website is configured safely — and most security tools either look scary or behave like attack tools.
>
> So I built Sentinel WebGuard: a passive, ethical website checkup.
>
> No attacking, no logins, no port scanning. It reads only public signals — TLS and HTTPS redirects, security headers (HSTS, CSP, X-Content-Type-Options), email-DNS trust (SPF, DMARC, MX), cookie safety and GDPR signals — and turns them into a 0–100 score with plain-language fixes a non-technical owner can actually act on.
>
> Every finding comes with the evidence, a clear explanation of the business impact, and one recommended fix. The principle I kept coming back to: explain impact first, and never overclaim.
>
> Building it taught me as much about communication as about security headers — a finding nobody understands is a finding nobody fixes.
>
> #Cybersecurity #WebSecurity #AppSec #BuildInPublic

### Post 3 — Sentinel Helpdesk AI
> Small businesses lose customers because messages get scattered across email, web forms, Instagram and phone notes — and answered late, or not at all.
>
> Sentinel Helpdesk AI is my take on fixing that. Every message becomes a tracked ticket. AI summarises the problem, assigns a priority, category and sentiment, and drafts a professional reply — but here's the part I'm most proud of: the owner reviews and approves every reply before it's sent.
>
> The biggest lesson from building it: make the safe path the default. Human-in-the-loop isn't a limitation, it's the feature. AI drafts; a person decides.
>
> Built with Next.js, TypeScript, Supabase and Zod validation on both client and server. It runs fully in demo mode with zero setup, and plugs into real AI/email providers when configured.
>
> #AI #SoftwareEngineering #FullStack #HumanInTheLoop

### Post 4 — Sentinel PortfolioOS (build-in-public + security depth)
> A list of GitHub links doesn't show how you think. So I built a portfolio that does — and made it a real platform.
>
> Sentinel PortfolioOS pairs a public, case-study-driven site with a private admin dashboard where I manage every project, skill and message without touching code.
>
> Under the hood I went defence-in-depth, because the product should model the engineering I talk about:
> • Shared Zod validation on the client AND re-validated on the server
> • Supabase Auth with Row-Level Security enforced at the database
> • A database audit-logging trigger that application code can't bypass
> • A hardened contact form: honeypot, rate limiting, and SHA-256 IP hashing (raw IPs never stored)
> • Safe escaped rendering — no untrusted HTML
>
> Shipped in TypeScript strict mode with tests and a green lint/typecheck/build.
>
> The thing I care about most: it never overclaims. Security is a set of trade-offs, and the docs say so.
>
> #Cybersecurity #SecureByDesign #NextJS #TypeScript #BuildInPublic
